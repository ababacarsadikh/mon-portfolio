1. Le microcontrôleur

Un microcontrôleur rassemble CPU, mémoire et entrées/sorties sur une seule puce. Il est conçu pour piloter un système embarqué de manière autonome.

ÉlémentRôle
CPUExécute les instructions du programme.
FlashConserve le sketch Arduino.
SRAMStocke les variables pendant l'exécution.
ADCConvertit une tension analogique en valeur numérique.
À retenir : toutes les connexions doivent partager une masse commune avec la carte Arduino.

2. Structure d'un programme Arduino

Un sketch Arduino utilise toujours un bloc de déclarations, une fonction setup() et une boucle loop().

Arduino C++
int led = 13;

void setup() {
  pinMode(led, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  digitalWrite(led, HIGH);
  delay(1000);
  digitalWrite(led, LOW);
  delay(1000);
}

3. LED & bouton poussoir

Une LED doit toujours être protégée par une résistance, généralement entre 220Ω et 330Ω sous 5V. Le bouton permet de lire une entrée numérique.

Bouton sur D8, LED sur D13
int led = 13;
int bouton = 8;

void setup() {
  pinMode(led, OUTPUT);
  pinMode(bouton, INPUT);
}

void loop() {
  int etat = digitalRead(bouton);
  digitalWrite(led, etat == HIGH ? HIGH : LOW);
}

Servo + potentiomètre
#include <Servo.h>

Servo monServo;
int pinPot = A0;

void setup() {
  monServo.attach(9);
}

void loop() {
  int valeur = analogRead(pinPot);
  int angle = map(valeur, 0, 1023, 0, 180);
  monServo.write(angle);
}

5. RFID RC522

Le module RC522 lit l'identifiant unique d'un badge via SPI. Il doit être alimenté en 3.3V, jamais en 5V.

Brochage courant : SDA→10, SCK→13, MOSI→11, MISO→12, RST→9, GND→GND, 3.3V→3.3V.
Lecture UID
#include <SPI.h>
#include <MFRC522.h>

#define RST_PIN 9
#define SS_PIN 10

MFRC522 rfid(SS_PIN, RST_PIN);

void setup() {
  Serial.begin(9600);
  SPI.begin();
  rfid.PCD_Init();
}

void loop() {
  if (!rfid.PICC_IsNewCardPresent()) return;
  if (!rfid.PICC_ReadCardSerial()) return;
  rfid.PICC_DumpToSerial(&(rfid.uid));
}
