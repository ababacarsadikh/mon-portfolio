Le Microcontrôleur

Un microcontrôleur est un circuit intégré (IC) qui rassemble sur une seule puce une unité de calcul, de la mémoire et des interfaces d'entrées/sorties. Compact et performant, il est au cœur de nombreux systèmes embarqués.

🌡️

Surveillance

Acquisition de capteurs : température, humidité, pression atmosphérique, luminosité…

🏠

Domotique

Contrôle des appareils électriques, volets, éclairage, chauffage à distance.

🤖

Robotique

Commande de robots industriels, servomoteurs, bras articulés, véhicules autonomes.

Composants internes d'un microcontrôleur
ComposantRôle
CPUUnité de calcul — exécute les instructions du programme
Flash (ROM)Mémoire programme non volatile — conserve le sketch
SRAMMémoire vive — variables et pile d'exécution
EEPROMMémoire persistante — données conservées après reset
Entrées/SortiesBroches numériques et analogiques pour interagir avec l'extérieur
Timers / PWMGénération de signaux temporisés et modulés en largeur
ADCConvertisseur analogique/numérique (0–5V → 0–1023)

La carte Arduino

Arduino est une carte électronique programmable open-source intégrant un microcontrôleur ATmega et un environnement de développement accessible. Elle existe en de nombreuses versions qui diffèrent par leur taille, puissance et nombre de broches.

Arduino Uno — Référence des broches
Broche(s)TypeDescription
0 → 13Numériques E/SHIGH (5V) ou LOW (0V) — pinMode, digitalRead, digitalWrite
A0 → A5Analogiques entréeValeur 0–1023 via analogRead()
3,5,6,9,10,11PWM (~)Sortie analogique simulée via analogWrite(0–255)
GNDMasseRéférence 0V pour tous les circuits
5VAlimentationTension stabilisée pour alimenter les composants
3.3VAlimentationPour les composants basse tension (ESP, modules SPI…)
VinEntrée alimentationTension externe non stabilisée (7–12V recommandé)
ℹ️

Les broches numériques 0 et 1 (RX/TX) sont utilisées par la communication série. Évitez de les utiliser en E/S pendant le débogage via le moniteur série.

3

Composants essentiels

La platine d'expérimentation (Breadboard)

Permet de réaliser des montages sans soudure. Chaque rangée de 5 trous est connectée en interne. Les rails rouge (+) et bleu (–) sont continus sur toute la longueur.

⚠️

Les deux moitiés de la breadboard sont séparées par un canal central. Ne pas connecter des composants à cheval sur ce canal sans un pont externe.

Les LED (Diodes électroluminescentes)

Une LED laisse passer le courant dans un seul sens : de l'anode (broche longue) vers la cathode (broche courte / méplat).

🔴

Résistance obligatoire ! Le courant de l'Arduino (40mA max/broche) est trop élevé pour une LED directe. Calculez la résistance : R = (Vcc − Vled) / Iled → typiquement 220Ω à 330Ω pour une LED rouge sur 5V.

Les Résistances

Une résistance limite la circulation du courant (mesurée en ohms Ω). Le code couleur sur le composant indique sa valeur. Les valeurs courantes en TP : 220Ω, 330Ω, 1kΩ, 10kΩ.

La connectique
  • Jumpers M-M (mâle-mâle) : entre deux points de la breadboard
  • Jumpers M-F (mâle-femelle) : entre Arduino et breadboard
  • Les connexions entre Arduino et composants passent toujours par une masse commune (GND)
Le circuit électrique de base

Une pile fournit une différence de potentiel : les électrons circulent du pôle − vers le pôle +. Si une LED est placée dans ce circuit avec sa résistance de protection, le passage des électrons génère de la lumière.

L'IDE Arduino

L'environnement de développement intégré (IDE) Arduino permet d'écrire, compiler et téléverser des programmes (sketches) directement sur la carte via USB.

Installation — étapes
  • Télécharger l'IDE depuis arduino.cc/en/software
  • Lancer l'installateur — cocher "Add Python to PATH" si demandé
  • Choisir le dossier d'installation et laisser progresser
  • Autoriser l'installation des pilotes USB (requis pour détecter la carte)
  • Cliquer Fermer une fois l'installation terminée

5

Programmation Arduino

Un sketch Arduino est structuré en trois blocs distincts. Cette structure est obligatoire — le compilateur la requiert.

📦

Bloc 1 — Déclarations

Variables globales, inclusion de bibliothèques (#include), définition des constantes et objets partagés entre les fonctions.

⚡

setup()

Exécuté une seule fois au démarrage. Configure les broches (pinMode), initialise les objets et la communication série.

🔄

loop()

Boucle infinie — le cœur du programme. Lit les capteurs, traite les données, commande les actionneurs, indéfiniment.

Fonctions essentielles
FonctionParamètre(s)Rôle
pinMode(pin, mode)pin, INPUT/OUTPUTConfigure une broche en entrée ou sortie
digitalWrite(pin, val)pin, HIGH/LOWÉcrit une valeur numérique sur une broche de sortie
digitalRead(pin)pinLit l'état HIGH ou LOW d'une broche d'entrée
analogRead(pin)A0–A5Lit une tension analogique → valeur 0–1023
analogWrite(pin, val)pin ~, 0–255Génère un signal PWM (simulation analogique)
delay(ms)millisecondesPause le programme (bloquant)
Serial.begin(baud)9600, 115200…Initialise la communication série
Serial.print(val)tout typeEnvoie une valeur vers le moniteur série
Structure complète d'un sketch
Arduino C++
// ── Bloc 1 : Déclarations ──────────────────────────────
int brocheLED = 13;  // LED intégrée sur la broche 13
int compteur = 0;

// ── Bloc 2 : setup() ──────────────────────────────────
void setup() {
  pinMode(brocheLED, OUTPUT);  // broche 13 en sortie
  Serial.begin(9600);           // moniteur série à 9600 baud
}

// ── Bloc 3 : loop() ───────────────────────────────────
void loop() {
  digitalWrite(brocheLED, HIGH);  // LED allumée
  delay(1000);                     // attendre 1 seconde
  digitalWrite(brocheLED, LOW);   // LED éteinte
  delay(1000);
  compteur++;
  Serial.println(compteur);       // afficher dans le moniteur
}
6

LED & Bouton poussoir

Exemple 1 — Clignotement (Blink)

La LED intégrée de la broche 13 clignote toutes les secondes. C'est le programme "Hello World" de l'Arduino.

Arduino C++
void setup() {
  pinMode(13, OUTPUT);
}
void loop() {
  digitalWrite(13, HIGH);  delay(1000);
  digitalWrite(13, LOW);   delay(1000);
}
Exemple 2 — LED commandée par bouton

Montage : bouton sur broche 8, LED sur broche 13 avec résistance 220Ω. La LED s'allume uniquement quand le bouton est appuyé.

Arduino C++
int brocheLED = 13;
int bouton = 8;
int etatbouton = 0;

void setup() {
  pinMode(brocheLED, OUTPUT);
  pinMode(bouton, INPUT);
}

void loop() {
  etatbouton = digitalRead(bouton);
  if (etatbouton == HIGH) {
    digitalWrite(brocheLED, HIGH);  // bouton appuyé → LED ON
  } else {
    digitalWrite(brocheLED, LOW);   // relâché → LED OFF
  }
}

🛠 TP1 — Feu tricolore

Réaliser un feu tricolore avec 3 LED qui tournent en boucle :

  • LED verte (broche 4) → allumée 3 secondes
  • LED orange (broche 3) → allumée 1 seconde
  • LED rouge (broche 2) → allumée 3 secondes
  • Séquence répétée indéfiniment tant que l'Arduino est alimenté

🛠 TP2 — Feu tricolore + Feu piéton

Feu tricolore (broches 10=vert, 11=orange, 12=rouge) + feu piéton bicolore (broche 9=rouge, 8=vert) avec bouton (broche 2) :

  • Fonctionnement normal identique au TP1
  • Feu piéton constamment rouge tant que le bouton n'est pas appuyé
  • Si bouton appuyé pendant le vert voiture → orange 1s puis rouge voiture 5s + vert piéton 5s
  • Après 5s : feu piéton repasse rouge, cycle normal reprend

Code exemple — Afficher "0" puis "9" en boucle
Arduino C++
int a=6, b=5, c=2, d=3, e=4, f=7, g=8;

void setup() {
  int broches[] = {a,b,c,d,e,f,g};
  for(int i=0; i<7; i++) pinMode(broches[i], OUTPUT);
}

void loop() {
  // Afficher "0"
  digitalWrite(a,HIGH); digitalWrite(b,HIGH); digitalWrite(c,HIGH);
  digitalWrite(d,HIGH); digitalWrite(e,HIGH); digitalWrite(f,HIGH);
  digitalWrite(g,LOW);
  delay(1000);

  // Afficher "9"
  digitalWrite(a,HIGH); digitalWrite(b,HIGH); digitalWrite(c,HIGH);
  digitalWrite(d,HIGH); digitalWrite(e,LOW);  digitalWrite(f,HIGH);
  digitalWrite(g,HIGH);
  delay(1000);
}

8

Entrées analogiques

Un signal analogique varie continûment dans le temps (température, lumière, son…). L'Arduino convertit ces valeurs en entiers via son CAN 10 bits : 0V → 0, 5V → 1023.

⚠️

Ne jamais dépasser 5V sur les broches analogiques sous peine de destruction définitive du microcontrôleur.

La photorésistance (LDR)

La résistance diminue quand la lumière augmente. Montée en diviseur de tension avec une résistance fixe (10kΩ), la tension mesurée sur A0 varie avec la luminosité.

Arduino C++ — TP4 : LED pilotée par lumière
const int BrochePhoto = A0;
const int BrocheLed  = 13;

void setup() {
  pinMode(BrocheLed, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  int lumiere = analogRead(BrochePhoto);
  Serial.println(lumiere);           // déboguer dans le moniteur série
  int seuil = 500;                    // ajuster selon l'ambiance
  if (lumiere > seuil) {
    digitalWrite(BrocheLed, LOW);    // trop lumineux → LED off
  } else {
    digitalWrite(BrocheLed, HIGH);   // sombre → LED on
  }
}
Le potentiomètre

Résistance variable rotative ou linéaire. Connecté entre 5V et GND avec le curseur sur A0, il génère une tension de 0 à 5V → valeur 0–1023. Utilisé pour contrôler la luminosité d'une LED via analogWrite().

Arduino C++ — Variation de luminosité
int BrocheLed = 9;  // broche PWM (~)
int BrocheAna = 0;

void setup() {
  pinMode(BrocheLed, OUTPUT);
}

void loop() {
  int val = analogRead(BrocheAna);       // 0–1023
  analogWrite(BrocheLed, val / 4);      // 0–255 pour PWM
}
💡

analogRead() retourne 0–1023 (10 bits). analogWrite() attend 0–255 (8 bits). D'où la division par 4 pour convertir l'une en l'autre.

Capteur DHT11 — Température & Humidité

Le DHT11 mesure la température (0–50°C ±2°C) et l'humidité relative (20–90% ±5%) sur un seul fil DATA. La conversion est effectuée sur 16 bits toutes les 2 secondes.

Brochage DHT11
BrocheNomConnexion
1VCC3.3V ou 5V
2DATABroche numérique Arduino (ex: 12) + résistance pull-up 10kΩ vers VCC
3NCNon connecté
4GNDGND
Installation de la bibliothèque
IDE Arduino
Outils → Gérer les bibliothèques → Rechercher "DHT sensor library" → Installer
Code complet — Lecture toutes les 2 secondes
Arduino C++
#include "DHT.h"
#define BrocheDHT 12
#define DHTTYPE   DHT11
DHT dht(BrocheDHT, DHTTYPE);

void setup() {
  Serial.begin(9600);
  Serial.println("Test capteur DHT11");
  dht.begin();
}

void loop() {
  delay(2000);
  float humidite    = dht.readHumidity();
  float temperature = dht.readTemperature();

  if (isnan(humidite) || isnan(temperature)) {
    Serial.println("Erreur lecture DHT !");
    return;
  }
  Serial.print("Humidité : ");    Serial.print(humidite, 1);    Serial.println(" %");
  Serial.print("Température : "); Serial.print(temperature, 1); Serial.println(" °C");
}

Écran LCD 16×2

L'écran LCD (Liquid Crystal Display) affiche 2 lignes de 16 caractères. Très utilisé dans les systèmes embarqués pour afficher des mesures ou messages sans ordinateur.

Code — Afficher un message sur 2 lignes
Arduino C++
#include "LiquidCrystal.h"

// LiquidCrystal(RS, E, D4, D5, D6, D7)
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

void setup() {
  lcd.begin(16, 2);           // 16 colonnes, 2 lignes
  lcd.print("Bonjour !");      // ligne 0, col 0 par défaut
}

void loop() {
  lcd.setCursor(0, 1);       // col 0, ligne 1
  lcd.print("Arduino !");
}
💡

La bibliothèque LiquidCrystal est incluse par défaut dans l'IDE Arduino. Pas d'installation supplémentaire nécessaire.

Capteur ultrason HC-SR04

Le HC-SR04 mesure des distances entre 2 cm et 4 m avec une précision de 3mm, via des ondes ultrasoniques (40 kHz, inaudibles). Tension d'alimentation : 5V.

Principe de fonctionnement
  • Envoyer une impulsion HIGH de 10µs sur la broche TRIGGER
  • Le capteur émet 8 impulsions ultrasoniques à 40kHz
  • Les ondes se propagent, touchent un obstacle et reviennent
  • La broche ECHO reste HIGH pendant la durée aller-retour
  • Distance = (durée × vitesse_son) / 2 — vitesse son ≈ 340 m/s
Brochage HC-SR04
Broche capteurConnexion Arduino
VCC5V
GNDGND
TRIGBroche 5 (OUTPUT)
ECHOBroche 6 (INPUT)
Code avec bibliothèque (simple)
Arduino C++
#include <HCSR04.h>
HCSR04 hc(5, 6);  // TRIG=5, ECHO=6

void setup() { Serial.begin(9600); }

void loop() {
  Serial.print("Distance : ");
  Serial.print(hc.dist());
  Serial.println(" cm");
  delay(200);
}
Code sans bibliothèque (calcul manuel)
Arduino C++
const byte TRIG = 5, ECHO = 6;
const float V_SON = 340.0 / 1000; // mm/µs

void setup() {
  Serial.begin(9600);
  pinMode(TRIG, OUTPUT); digitalWrite(TRIG, LOW);
  pinMode(ECHO, INPUT);
}

void loop() {
  // Impulsion TRIGGER 10µs
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);

  // Mesure durée écho
  long duree = pulseIn(ECHO, HIGH, 25000UL);
  float dist_mm = duree / 2.0 * V_SON;

  Serial.print("Distance: ");
  Serial.print(dist_mm / 10.0, 1);
  Serial.println(" cm");
  delay(500);
}

Servomoteur

Un servomoteur positionne son axe à un angle précis (0°–180°) et le maintient. Très utilisé en robotique, modélisme, domotique. Il est contrôlé par un signal PWM à période de 20ms.

Relation largeur d'impulsion → angle
Durée impulsionPosition
1 ms0°
1.5 ms90° (centre)
2 ms180°
Code — Balayage automatique 0° → 180° → 0°
Arduino C++
#include <Servo.h>
Servo monServo;

void setup() {
  monServo.attach(9);  // broche PWM signal
}

void loop() {
  for (int pos = 0; pos <= 180; pos++) {
    monServo.write(pos);
    delay(15);
  }
  for (int pos = 180; pos >= 0; pos--) {
    monServo.write(pos);
    delay(15);
  }
}
Code — Contrôle par potentiomètre
Arduino C++
#include <Servo.h>
Servo monServo;
int pinServo = 9;
int pinPot   = A0;

void setup() { monServo.attach(pinServo); }

void loop() {
  int valPot = analogRead(pinPot);          // 0–1023
  int angle  = map(valPot, 0, 1023, 0, 180); // conversion
  monServo.write(angle);
}
💡

La fonction map(val, 0, 1023, 0, 180) convertit linéairement une valeur d'une plage vers une autre. Très utile pour adapter des lectures analogiques à des plages d'angles ou de vitesses.

Clavier numérique (Keypad)

Un clavier matriciel (3×4 ou 4×4) fonctionne par balayage de lignes et colonnes. Chaque touche relie une ligne à une colonne. Bibliothèque Keypad requise.

Brochage clavier 4×4 → Arduino
LigneBrocheColonneBroche
R13C17
R22C26
R31C35
R40C44
Arduino C++
#include <Keypad.h>

const byte ROWS = 4, COLS = 4;
char keys[ROWS][COLS] = {
  {'1','2','3','A'},
  {'4','5','6','B'},
  {'7','8','9','C'},
  {'*','0','#','D'}
};
byte rowPins[ROWS] = {3,2,1,0};

    Serial.println(touche);
  }
}

RFID RC522

La technologie RFID (Radio Frequency IDentification) permet d'identifier sans contact un badge ou tag via des ondes radio. Le module RC522 émet des ondes électromagnétiques et lit l'UID unique de chaque tag (96 bits, standard EPC).

Applications
  • Contrôle d'accès par badge
  • Suivi de marchandises en logistique
  • Bibliothèques, parking, pointage
Brochage RC522 → Arduino Uno (SPI)
RC522Arduino
SDA (SS)10
SCK13
MOSI11
MISO12
RST9
GNDGND
3.3V3.3V ← Ne pas mettre 5V !
Code — Lire l'UID d'un badge
Arduino C++
#include "SPI.h"
#include "MFRC522.h"
#define RST_PIN 9
#define SS_PIN  10
MFRC522 mfrc522(SS_PIN, RST_PIN);

void setup() {
  Serial.begin(9600);
  SPI.begin();
  mfrc522.PCD_Init();
  Serial.println("Approcher un badge...");
}

void loop() {
  if (!mfrc522.PICC_IsNewCardPresent()) return;
  if (!mfrc522.PICC_ReadCardSerial())  return;
  mfrc522.PICC_DumpToSerial(&(mfrc522.uid));
}
Code — Contrôle d'accès par badge autorisé
Arduino C++
#include "SPI.h"
#include "MFRC522.h"
#define RST_PIN 9
#define SS_PIN  10

byte readCard[4];
String cardID = "20C3935E";  // remplacer par votre UID
String tagID  = "";
MFRC522 mfrc522(SS_PIN, RST_PIN);

void setup() {
  Serial.begin(9600);
  SPI.begin(); mfrc522.PCD_Init();
  pinMode(5, OUTPUT);  // LED rouge (accès refusé)
  pinMode(6, OUTPUT);  // LED verte (accès autorisé)
}

boolean getID() {
  if (!mfrc522.PICC_IsNewCardPresent()) return false;
  if (!mfrc522.PICC_ReadCardSerial())  return false;
  tagID = "";
  for (uint8_t i=0; i<4; i++) tagID.concat(String(mfrc522.uid.uidByte[i], HEX));
  tagID.toUpperCase();
  mfrc522.PICC_HaltA();
  return true;
}

void loop() {
  while (getID()) {
    if (tagID == cardID) {
      Serial.println("Accès autorisé !");
      digitalWrite(6, HIGH);  // LED verte ON
    } else {
      Serial.println("Accès refusé !");
      digitalWrite(5, HIGH);  // LED rouge ON
    }
    Serial.print("UID: "); Serial.println(tagID);
    delay(2000);
    digitalWrite(5, LOW); digitalWrite(6, LOW);
  }
}
⚠️

Le module RC522 fonctionne en 3.3V. Le connecter à 5V le détruirait. Vérifiez soigneusement l'alimentation avant de brancher.

✅

Pour récupérer votre UID, téléversez d'abord le code de lecture basique, ouvrez le moniteur série, approchez votre badge et notez l'UID affiché. Remplacez ensuite la valeur de cardID dans le code de contrôle d'accès.
